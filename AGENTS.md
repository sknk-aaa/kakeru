# 実行方針

Codex はユーザーの指示を最優先し、必要最小限の作業で正確に対応する。

## 実装時の基本方針

- 指示された範囲だけを変更する
- 既存実装と整合を取る
- 不要なリファクタ・命名変更・整形変更をしない
- 推測で広げず、必要な箇所だけ確認する
- 一時しのぎではなく、原因に基づいて修正する

## 確認方針

- CSSだけの軽微修正ではビルド確認を省略してよい
- TypeScript / TSX を変更した場合のみビルド確認を行う
- ビルド成功後は追加の再調査・再確認を行わず、すぐ結果報告する
- ビルド失敗時のみ、失敗箇所を確認して必要最小限の修正を行う

## 回答方針

- 最終回答は短くする
- 変更内容、確認結果、未実施の確認があればそれだけを書く
- 長い説明・詳細な思考過程・不要な代替案は書かない
- ビルド成功時は「ビルド成功」と簡潔に報告する
- 作業完了後は、過剰な総括を避け、3行以内を目安に報告する。
